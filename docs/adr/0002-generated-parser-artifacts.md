# ADR 0002: Do Not Track Generated Parser Artifacts

## Status

Accepted

## Context

`src/parser.c` is generated by `tree-sitter generate` and is large enough to
make ordinary pull request review and merge checks unreliable. Keeping it in
Git causes huge diffs for small grammar changes and makes the review surface
less useful.

The generated parser is still required for native binding builds and npm
packages.

## Decision

Do not track `src/parser.c` or `src/grammar.json` in Git. Keep tracking
`src/node-types.json` because it is compact enough to review and documents the
public AST shape consumed by downstream tools.

Generate parser artifacts in CI before tests and during npm packaging through
`prepack`. Keep `src/**` in the npm package file list so generated artifacts are
included in published tarballs.

## Consequences

Grammar pull requests focus on `grammar.js`, corpus tests, queries,
documentation, and `src/node-types.json` instead of large generated C diffs.

Fresh checkouts must run `pnpm generate` before building native bindings
directly. Existing scripts that run the full test flow already generate the
parser before binding tests.

Published npm packages still contain the generated parser artifacts needed by
consumers that install from the package tarball.
